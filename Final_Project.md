# Proyecto Final: Aplicación de Métodos de Pruebas con Cypress

## Objetivo del Proyecto
El objetivo de este proyecto final es permitir a los estudiantes aplicar lo aprendido sobre pruebas de aplicaciones web utilizando Cypress. Los estudiantes deberán desarrollar una suite de pruebas para una aplicación web de su elección, utilizando las técnicas y conceptos aprendidos durante el curso.

## Requisitos del Proyecto

1. **Selección de la Aplicación:**
   - Elegir una aplicación web en la que se deseen implementar pruebas. Esto puede ser una aplicación personal, un proyecto de grupo o una aplicación de código abierto.

2. **Configuración de Cypress:**
   - Instalar Cypress en el proyecto seleccionado. Asegúrate de que todas las dependencias necesarias se instalen correctamente y que la aplicación esté lista para ejecutar pruebas.

3. **Escritura de Pruebas:**
   - **Pruebas unitarias**: Escribir al menos cinco pruebas unitarias que verifiquen la funcionalidad básica de la aplicación.
   - **Pruebas de integración**: Asegurarse de que las pruebas cubran la integración entre diversos módulos de la aplicación.
   - **Pruebas end-to-end (e2e)**: Implementar al menos tres escenarios de prueba end-to-end que simulen la experiencia del usuario en la aplicación.

4. **Uso de BDD (Desarrollo Guiado por Comportamiento):**
   - Implementar casos de prueba utilizando el enfoque BDD con Cucumber y Gherkin, creando al menos dos archivos `.feature` que describan flujos de trabajo de usuario.

5. **Integración CI/CD:**
   - Configurar un pipeline de CI/CD utilizando GitHub Actions:
     - Incluir pasos para instalar dependencias, ejecutar pruebas y construir la aplicación.
     - Asegurarse de que el pipeline se ejecute correctamente en cualquier nuevo push al repositorio.

6. **Documentación:**
   - Documentar el proceso de pruebas en un archivo `README.md` donde se incluya:
     - Instrucciones de instalación y ejecución.
     - Descripción de las pruebas realizadas.
     - Resultados de las pruebas, incluyendo capturas de pantalla o videos si es posible.

## Entregables
Los estudiantes deberán entregar lo siguiente:
- El repositorio de su aplicación con las pruebas en un servicio como GitHub o GitLab.
- Un video corto (5-10 minutos) donde demuestren cómo corren las pruebas en Cypress.
- Un documento que explique el enfoque de pruebas utilizado, los desafíos encontrados y cómo los resolvieron.

## Criterios de Evaluación
- **Completitud**: Se evaluará si se cumplen todos los requisitos del proyecto.
- **Calidad de las Pruebas**: Se evaluará la efectividad y la calidad de las pruebas escritas.
- **Documentación**: La claridad y la exhaustividad de la documentación presentada.
- **Presentación**: La claridad y efectividad de la presentación del video.

¡Buena suerte y disfruten del proceso de prueba!
